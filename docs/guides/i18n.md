# 🌐 Internationalization (i18n) Guide

**Last updated:** 2026-07-22

---

## 📋 Overview

TEA Group website supports **2 locales**:
- **Vietnamese (vi)** - Default locale
- **English (en)** - Available via `/en` route

Uses **next-intl** for i18n in Next.js 15.

---

## 🎯 Supported Locales

| Code | Language | Default | Region |
|------|----------|---------|--------|
| `vi` | Vietnamese | ✅ Yes | Vietnam (Ho Chi Minh City) |
| `en` | English | No | International |

**Default locale:** `vi` (Vietnamese)

---

## 🔧 URL Structure

```
/                    → Redirects to /vi
/vi                  → Vietnamese homepage
/en                  → English homepage
/vi/solutions        → Vietnamese solutions page
/en/solutions        → English solutions page
/vi/admin            → Vietnamese admin (protected)
```

## 🔧 Usage Patterns

### In Server Components (ASYNC)
```typescript
import { getTranslations } from 'next-intl/server';

export default async function HomePage() {
  const t = await getTranslations('Home');
  
  return (
    <div>
      <h1>{t('title')}</h1>
      <p>{t('description')}</p>
    </div>
  );
}
```

### In Client Components
```typescript
'use client';
import { useTranslations } from 'next-intl';

export function Component() {
  const t = useTranslations('Home');
  
  return (
    <div>
      <h1>{t('title')}</h1>
    </div>
  );
}
```

### With Parameters
```json
// locales/vi/home.json
{
  "greeting": "Xin chào {name}",
  "itemCount": "{count} items"
}
```

```typescript
const t = await getTranslations('Home');
const message = t('greeting', { name: 'TEA Group' });
// → "Xin chào TEA Group"

const count = t('itemCount', { count: 5 });
// → "5 items" (pluralized)
```

## 🔗 Locale-Aware Navigation

### Link Component
```typescript
import { Link } from '@/i18n/navigation';

// Locale-aware - automatically adds current locale
<Link href="/solutions">Solutions</Link>
// → /vi/solutions (if on /vi page)
// → /en/solutions (if on /en page)
```

### useRouter Hook
```typescript
import { useRouter } from '@/i18n/navigation';

const router = useRouter();
router.push('/solutions');
// → Automatically uses current locale
```

### External Links
```typescript
// Regular Next.js Link for external links
import Link from 'next/link';

<Link href="https://example.com">External</Link>
```

## ⚠️ Critical Gotchas

### 1. Async Server Components
```typescript
// ❌ WRONG - useTranslations in async server component
export default async function Page() {
  const t = useTranslations(); // 💥 CRASHES!
}

// ✅ RIGHT - getTranslations in async server component
export default async function Page() {
  const t = await getTranslations(); // ✅ Works!
}
```

### 2. next-intl Bug (v3.26.5)
```typescript
// ❌ WRONG - hasLocale not exported
import { hasLocale } from 'next-intl'; // 💥 Undefined!

// ✅ RIGHT - Use local isLocale
import { isLocale } from '@/i18n/routing';
```

### 3. Locale Detection Order
```typescript
// Detection priority:
1. URL path (/vi or /en)
2. Cookie preference
3. Accept-Language header
4. Defaults to 'vi'
```

### Per-locale Routes
```typescript
// app/[locale]/page.tsx
// → Serves both /vi and /en

// app/[locale]/vi-only/page.tsx
// → Only for Vietnamese content
```

### Conditional Rendering
```typescript
const t = await getTranslations();
const locale = await getLocale();

if (locale === 'vi') {
  return <VietnameseSpecificContent />;
}
return <EnglishContent />;
```

## 🔧 Configuration

### i18n Config (apps/web/src/i18n/config.ts)
```typescript
export const locales = ['vi', 'en'] as const;
export const defaultLocale = 'vi' as const;
export const localePrefix = 'always'; // Always show locale in URL
```

### next-intl Config (apps/web/src/i18n/request.ts)
```typescript
export default getRequestConfig(async ({ locale }) => {
  return {
    messages: (await import(`../../locales/${locale}.json`)).default
  };
});
```
